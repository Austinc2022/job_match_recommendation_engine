It reads data from CSV files containing information about jobseekers and jobs, computes the skill matches, and outputs a sorted list of job recommendations for each jobseeker.</p> <h2 id="features">Features</h2> <ul> <li><strong>Skill Matching</strong>: Calculates the number of matching skills between jobseekers and jobs.</li> <li><strong>Percentage Match</strong>: Computes the percentage of matching skills relative to the job's required skills.</li> <li><strong>Flexible Input</strong>: Allows specifying custom CSV files for jobseekers and jobs.</li> <li><strong>Jobseeker Filtering</strong>: Generates recommendations for all jobseekers or a specific jobseeker by ID or name.</li> <li><strong>Detailed Output</strong>: Includes matched and unmatched skills in the output as separate arrays.</li> <li><strong>Sorted Recommendations</strong>: Outputs recommendations sorted by jobseeker ID, matching skill percentage in descending order, and job ID in ascending order.</li> <li><strong>Modular Design</strong>: Organized codebase with separate modules for data models, utilities, and recommendation logic.</li> <li><strong>Unit Tests</strong>: Comprehensive unit tests for each module to ensure code reliability.</li> </ul> <h2 id="project-structure">Project Structure</h2> <pre> job_match_recommendation_engine/<br> ├── README.md<br> ├── requirements.txt<br> ├── tests/<br> ├── __init__.py<br> ├── test_job.py<br> ├── test_jobseeker.py<br> ├── test_recommender.py<br> └── test_utils.py<br> </pre> <ul> <li><code>main.py</code>: The entry point of the application.</li> <li><code>jobmatch/</code>: Contains modules for the application's functionality. <ul> <li><code>job.py</code>: Defines the <code>Job</code> class.</li> <li><code>jobseeker.py</code>: Defines the <code>Jobseeker</code> class.</li> <li><code>recommender.py</code>: Contains the <code>Recommender</code> class.</li> <li><code>utils.py</code>: Provides utility functions for reading data.</li> </ul> </li> <li><code>data/</code>: Stores the input CSV files.</li> <li><code>tests/</code>: Contains unit tests for each module.</li> </ul> <h2 <pre><code>python main.py</code></pre> <h3 id="command-line-arguments">Command-Line Arguments</h3> <ul> <li><code>--jobseekers</code>: Path to the jobseekers CSV file (default: for a jobseeker by name</strong>: <pre><code>python main.py --jobseeker_name "Bob Applicant"</code></pre> </li> </ul> <h2 id="output-explanation">Output Explanation</h2> <p>The program outputs a CSV-formatted list with the following columns:</p> <ul> <li><strong><code>jobseeker_id</code></strong>: ID of the jobseeker.</li> <li><strong><code>jobseeker_name</code></strong>: Name of the jobseeker.</li> <li><strong><code>job_id</code></strong>: ID of the job.</li> <li><strong><code>job_title</code></strong>: Title of the job.</li> <li><strong><code>matching_skill_count</code></strong>: Number of skills that match between the jobseeker and the job.</li> <li><strong><code>matching_skill_percent</code></strong>: Percentage of matching skills relative to the job's required skills.</li> <li><strong><code>matched_skills</code></strong>: Array of skills that the jobseeker has which are required by the job.</li> <li><strong><code>unmatched_skills</code></strong>: Array of required skills that the jobseeker does not have.</li> </ul> <p><strong>Notes</strong>:</p> <ul> <li>Matched and unmatched skills are displayed as JSON-formatted arrays within the CSV output.</li> <li>Empty arrays are represented as <code>[]</code>.</li> </ul> <p><strong>Sample Output</strong>:</p> <pre><code>jobseeker_id,
